폰트 프로필

폰트도 생성되는 UI의 일부입니다.

스킬은 폰트명, CSS/다운로드 URL, 대체 폰트 스택, 사용 이유를 함께 제안합니다. 확인된 브랜드 폰트가 있으면 우선 사용하고, 없으면 무료 한국어 UI 폰트를 안전하게 선택합니다.

기본

Pretendard

핀테크, 커머스, 생산성 등 대부분의 한국 모바일 UI에 무난한 기본 폰트입니다.

브리프 필드 예시 
font_profile:

  css_url: "https://cdn.jsdelivr.net/.../pretendard.css"
  fallback: "-apple-system, BlinkMacSystemFont, Apple SD Gothic Neo, Noto Sans KR, sans-serif"
  confidence: "recommended fallback"
  reason: "Neutral Korean mobile UI with strong numeric readability"

레퍼런스 지식 베이스

단순 데모가 아니라 화면 분석에서 뽑은 패턴 지식을 반영합니다.

공개 스킬에는 택소노미, 컴포넌트 상태, 패턴 커버리지, 에셋 역할, 안티패턴, 전달 품질을 일반화한 레퍼런스만 포함합니다. 출처 자료는 공개 파일에 넣지 않습니다.

01

패턴 분석 인사이트

references/pattern-analysis-insights.md

모바일 UI 화면 분석에서 뽑은 공개-safe 개선사항: 부족한 앱 타입, 상태 처리, 에셋 역할, 반복 안티패턴.

레퍼런스 열기
02

택소노미 필터 모델

references/taxonomy-filter-model.md

앱 종류와 UI 패턴을 분리해 예시와 생성 스펙을 두 축으로 계속 확장할 수 있게 합니다.

레퍼런스 열기
03

모바일 패턴 라이브러리

references/mobile-pattern-library.md

컴포넌트, 상태, 인터랙션, 카피 요구사항, 접근성 메모, 안티패턴을 포함한 재사용 모바일 패턴입니다.

레퍼런스 열기
04

품질 리뷰 체크리스트

references/quality-review-checklist.md

패턴 적합성, 시각 시스템, 인터랙션 상태, 접근성, 구현 전달 품질을 배포 전 점검합니다.

레퍼런스 열기

스킬이 생성하는 것

AI 디자인/코드 에이전트에 명확하게 전달되도록 설계했습니다.

01

패턴 시스템

패턴 그룹, 레이아웃 아키타입, 컴포넌트 목록, 인터랙션 모델, 상태 매트릭스.

02

화면 브리프

목표, 사용자 의도, 위계, 섹션, 고정/스티키 영역, 카피 방향.

03

언어 모드

영어 또는 한국어를 선택합니다. 요청하지 않으면 두 언어를 섞지 않습니다.

04

폰트 프로필

추천 폰트, CSS URL, fallback stack, 라이선스 확인 메모.

05

구현 전달문

코딩 에이전트, Figma 에이전트, 디자인 시스템 구현용 프롬프트.

06

모바일 제약

안전 영역, 터치 타깃, 고정 CTA, 키보드 동작, 접근성.
