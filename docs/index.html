Korean quick summary

한국 사용자도 쉽게 이해할 수 있게.

이 스킬은 모바일 UI를 막연히 “그려줘”라고 하는 대신, 로컬에 준비된 레퍼런스 메타데이터를 바탕으로 화면 유형, 컴포넌트, 에셋 단서, 품질 체크 기준을 정리해 에이전트에게 전달합니다.

무엇을 하나요?

회원가입, 본인인증, 결제, 상품상세, 지도 예약, 피드 같은 모바일 화면을 만들기 위한 구조화된 브리프를 생성합니다.

무엇을 공개하나요?

공개 레포에는 스킬 설명, 샘플 브리프, HTML/CSS 데모만 포함합니다. 원본 이미지와 private 데이터는 포함하지 않습니다.

어떻게 쓰나요?

에이전트에게 원하는 서비스/화면을 말하면, 관련 화면 유형과 컴포넌트 기준을 정리한 Markdown 브리프와 JSON 스펙을 만듭니다.

Font profiles

Typography belongs in the UI brief too.

Each generated brief can include a font profile: confirmed brand font when known, or a safe free Korean font fallback when the exact service font is unknown.

서비스마다 폰트가 다르기 때문에, 실제 UI 브리프에는 폰트명, 다운로드/CSS 주소, fallback stack, 사용 이유를 함께 넣도록 설계했습니다.

Pipeline

From a private reference corpus to agent-ready specs.

01

Private reference import

Uses locally authorized reference metadata. Raw sources, credentials, and provider connectors are not published.

02

Image/media index

Builds a metadata-only JSONL index before downloading. Avoids repeated API scans.

03

Screen taxonomy

Normalizes real pattern names into signup, payment, PDP, map, chat, settings, and more.

04

Asset extraction

Tracks icons, splash images, crops, MP4/SVG/JSON media metadata, and color palettes.

05

Brief compiler

Writes Markdown and JSON specs with reference evidence, assets, native constraints, and copy guidance.

06

Quality gates

Scores integrity, coverage, prompt completeness, reference relevance, and asset fit.

Quality, not volume

Checks whether references will actually improve the UI.

For multi-step prompts like 회원가입과 결제, the retriever detects multiple target screens and excludes unrelated evidence if relevant rows exist.

단순히 화면 수를 늘리는 것이 아니라, 요청한 화면에 실제로 도움이 되는 레퍼런스인지 검사합니다.
