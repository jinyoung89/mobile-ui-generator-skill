# Domain guides

Use the category to tune trust, density, copy, and error recovery; keep the
numeric layout contract shared.

| Domain | Emphasize | Avoid |
| --- | --- | --- |
| Finance / identity | balance clarity, confirmation, audit-friendly copy | decorative ambiguity, hidden fees |
| Commerce / marketplace | product confidence, address, totals, recovery | surprise charges, weak purchase feedback |
| Mobility / maps | location permission, route context, sheet states | map-only actions, unsafe edge controls |
| Healthcare | privacy, consent, readable records, support | diagnosis certainty, dense tiny text |
| Education / games | progress, motivation, feedback, pause/retry | reward-only hierarchy, motion dependence |
| Messaging / media | thread continuity, composer, upload progress | keyboard occlusion, lost drafts |
| Travel / tickets | itinerary, time zone, QR fallback, offline state | hidden gate conditions |
| AI / productivity | transparent progress, edit/apply, undo, review | irreversible generation, unexplained waiting |
| Support / utilities | search, clear status, contact path, receipts | dead-end errors, jargon-only copy |

When a domain is rare or ambiguous, explain the product rationale and keep the
pattern choice explicit in metadata. Do not imply that a generic pattern is a
copy of a particular product.
